VC Events — Google Sheet Setup

How to format your spreadsheet and connect it to the FC Events app for one-click syncing.

How it works

Google Sheets can publish any tab as a live CSV URL. The app fetches that URL, parses the rows, and runs them through the same import flow — no OAuth, no API keys required. Any time you update the sheet and re-sync in the app, it picks up the latest data without touching photos or any fields not in your sheet.

Publishing your sheet — step by step

1

Create or open your Google Sheet

Row 1 must be your column headers — exact key names from the table above (all lowercase, underscores). Column order doesn't matter. You can include any subset of columns.

2

Fill in your event data starting at row 2

Every row needs a slug — this is how the importer finds the matching WordPress post. The slug is the URL-friendly name, e.g. crssd-fall-2026. Dates must be YYYY-MM-DD format.

3

Publish the sheet as CSV

Go to File → Share → Publish to web. In the dialog, select the specific tab (sheet) you want to publish, then change the format dropdown from "Web page" to Comma-separated values (.csv). Click Publish and confirm.

⚠️ Make sure you select the specific tab, not "Entire document" — otherwise the URL won't output clean CSV.
4

Copy the published URL

After publishing, Google gives you a URL. It looks like this:

https://docs.google.com/spreadsheets/d/YOUR_SHEET_ID/pub?gid=0&single=true&output=csv
The gid number identifies which tab. If you have multiple tabs, each will have a different gid in its URL.
5

Paste the URL into the app

In the FC Events app → Events → ⬆ Import → "Sync from Google Sheet" card. Paste the URL and tap Fetch & Preview. You'll see the same preview table as a file upload. Tap Import when ready.

✓ The URL is remembered in the app so future syncs are one tap.

Notes & gotchas

Photos are safe. The importer only writes fields that are present in your sheet. Photos, logos, lineups, and anything else stored in WordPress stay untouched.
Slug must match exactly. If a row's slug doesn't match an existing WP post, it's skipped (not created). To create new events, the post must exist in WordPress first.
Sheet must be public. "Publish to web" is different from "Share." The sheet itself can stay private — only the published CSV URL needs to be publicly accessible for the app to fetch it.
Dates in Google Sheets. Format the date columns as Plain Text (not Date) to prevent Sheets from auto-converting to MM/DD/YYYY. The importer expects YYYY-MM-DD.
